## 카데인 알고리즘이란?

카데인 알고리즘은 배열에서 **연속된 부분 배열의 최대 합**을 구하는 알고리즘이다.

예를 들어 아래 배열이 있다고 하자.

```text
[-2, 1, -3, 4, -1, 2, 1, -5, 4]
```

이 배열에서 최대 합을 만드는 연속 부분 배열은 다음과 같다.

```text
[4, -1, 2, 1]
```

합은 `6`이다.

단순하게 모든 부분 배열을 확인하면 시간 복잡도가 `O(N^2)` 이상이 될 수 있다.
하지만 카데인 알고리즘을 사용하면 배열을 한 번만 순회해서 `O(N)`에 답을 구할 수 있다.

## 핵심 아이디어

각 위치에서 생각할 것은 하나다.

> 현재 원소를 이전 부분 배열에 이어 붙일 것인가, 아니면 현재 원소부터 새로 시작할 것인가?

현재 위치의 값을 `num`이라고 하면, 현재 위치에서 끝나는 최대 부분 배열의 합은 다음 둘 중 큰 값이다.

```text
current = max(num, current + num)
```

- `num`: 이전까지의 합을 버리고 현재 원소부터 새로 시작
- `current + num`: 이전 부분 배열에 현재 원소를 이어 붙임

그리고 전체 정답은 매 순간의 `current` 중 가장 큰 값이다.

```text
answer = max(answer, current)
```

## 예시로 보기

배열이 다음과 같다고 하자.

```text
[-2, 1, -3, 4, -1, 2, 1, -5, 4]
```

처음 값 `-2`에서 시작한다.

| num | current 계산 | current | answer |
| --- | --- | ---: | ---: |
| -2 | 시작값 | -2 | -2 |
| 1 | max(1, -2 + 1) | 1 | 1 |
| -3 | max(-3, 1 - 3) | -2 | 1 |
| 4 | max(4, -2 + 4) | 4 | 4 |
| -1 | max(-1, 4 - 1) | 3 | 4 |
| 2 | max(2, 3 + 2) | 5 | 5 |
| 1 | max(1, 5 + 1) | 6 | 6 |
| -5 | max(-5, 6 - 5) | 1 | 6 |
| 4 | max(4, 1 + 4) | 5 | 6 |

최종 답은 `6`이다.

## Java 코드

```java
public class Kadane {
    public static int maxSubArray(int[] nums) {
        int current = nums[0];
        int answer = nums[0];

        for (int i = 1; i < nums.length; i++) {
            current = Math.max(nums[i], current + nums[i]);
            answer = Math.max(answer, current);
        }

        return answer;
    }

    public static void main(String[] args) {
        int[] nums = {-2, 1, -3, 4, -1, 2, 1, -5, 4};
        System.out.println(maxSubArray(nums)); // 6
    }
}
```

## 왜 음수 배열도 처리할 수 있을까?

배열이 전부 음수인 경우도 있다.

```text
[-5, -2, -8]
```

이때 최대 부분 배열의 합은 `-2`다.

그래서 `current`와 `answer`를 `0`으로 시작하면 안 된다.
처음 값을 `nums[0]`으로 시작해야 모든 원소가 음수인 경우도 올바르게 처리할 수 있다.

```java
int current = nums[0];
int answer = nums[0];
```

## 복잡도

- 시간 복잡도: `O(N)`
- 공간 복잡도: `O(1)`

배열을 한 번만 순회하고, 추가 배열을 만들지 않기 때문에 효율적이다.

## 정리

카데인 알고리즘은 DP(Dynamic Programming)의 관점에서 볼 수 있다.

현재 위치에서의 최선은 다음 둘 중 하나다.

- 이전 합에 현재 값을 이어 붙인다.
- 이전 합을 버리고 현재 값부터 새로 시작한다.

이 선택을 매 위치마다 반복하면, 연속 부분 배열의 최대 합을 선형 시간에 구할 수 있다.
